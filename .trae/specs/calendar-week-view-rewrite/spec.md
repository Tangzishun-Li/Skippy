# 日历周视图重写规格

## 问题描述
当前周视图存在以下问题：
1. 周六（最后一列）看不见/丢失
2. 每列宽度不一致
3. 时间列和各天列宽度不匹配
4. 课程事件没有跨多个时间段显示

## 变更内容
- 重新设计周视图的渲染逻辑
- 确保 8 列网格正确显示（1 列时间 + 7 列星期）
- 课程事件应根据起止时间跨多行显示

## 影响范围
- 受影响组件：日历周视图
- 修改文件：
  - `src/renderer/js/core/calendar.js` - renderWeekView 函数
  - `src/renderer/css/components/calendar.css` - 周视图样式

## 需求说明
### 需求：周视图正确显示
系统必须正确显示周视图，包含：
- 第一列固定宽度显示时间（50px）
- 其他 7 列平均分配剩余宽度显示周日到周六
- 所有行高度一致

#### 场景：显示当前周的日历
- **当** 用户切换到周视图
- **则** 显示从周日到周六的 7 天 + 时间列
- **则** 第一列为时间列（固定 50px 宽）
- **则** 其他 7 列为日期列（等宽）

### 需求：课程事件跨时段显示
如果课程时间跨越多个小时，事件应该横跨多行

#### 场景：课程 9:00-12:00
- **当** 课程时间为 9:00-12:00
- **则** 事件应在 9:00、10:00、11:00 三个时间段上显示
- **则** 事件高度应覆盖 3 个时间格（每个 50px，共 150px）

## 技术方案
1. 使用 CSS Grid 布局，8 列 18 行
2. JS 动态设置每个单元格的 grid-row 和 grid-column
3. 课程事件渲染时计算持续时间，设置 grid-row-span
